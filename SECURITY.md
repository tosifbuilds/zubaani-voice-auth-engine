# Security Policy

At Zubaani, security and accessibility go hand in hand. We take the security of our voice-first authentication engine and our users' data very seriously. 

## Supported Versions

We actively provide security updates for the latest major versions of the Zubaani engine.

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover any security-related issues, vulnerabilities in the Firebase authentication flow, or AI voice latency exploits, **please do not report them through public GitHub issues.**

Instead, please report it privately to our core maintainer team by emailing:
**[tausifalam7783@gmail.com]**

### What to include in your report:
* A detailed description of the vulnerability.
* The steps required to reproduce the issue.
* Potential impact of the vulnerability.

### Our Commitment:
* We will acknowledge receipt of your vulnerability report within 48 hours.
* We will provide a timeline for addressing the issue and keep you updated on the patching process.
* Once the issue is resolved, we will credit your contribution (if desired) in our security advisories.

Thank you for helping keep the open-source ecosystem safe.
